# 分散システム系

アプリケーションやサービスをどの単位で分割し、どのように通信・配置するかを扱うカテゴリです。

## 作成済みドキュメント

| アーキテクチャ | 概要 |
| --- | --- |
| [モノリシックアーキテクチャ](./モノリシックアーキテクチャ.md) | 主要機能を1つのアプリケーション、1つのデプロイ単位としてまとめる構成です。 |
| [モジュラーモノリス](./モジュラーモノリス.md) | 単一デプロイのまま、内部を明確なモジュール境界で分割する構成です。 |
| [マイクロサービスアーキテクチャ](./マイクロサービスアーキテクチャ.md) | システムを小さな独立サービスに分け、それぞれを独立して開発、デプロイ、運用する構成です。 |
| [SOA](./SOA.md) | 機能をサービスとして提供し、標準化された契約を通じて複数システムを連携する設計です。 |
| [サーバーレスアーキテクチャ](./サーバーレスアーキテクチャ.md) | サーバー管理をクラウド事業者に委ね、関数やマネージドサービスを組み合わせる構成です。 |
| [イベント駆動アーキテクチャ](./イベント駆動アーキテクチャ.md) | 状態変化や出来事をイベントとして発行し、購読側が非同期に反応する構成です。 |
| [Client-Server Architecture](./クライアントサーバーアーキテクチャ.md) | 利用者に近いクライアントと、処理やデータを提供するサーバーに責務を分ける基本構成です。 |
| [Peer-to-Peer Architecture](./ピアツーピアアーキテクチャ.md) | 中央サーバーに依存せず、ノード同士が対等に通信しリソースを共有する構成です。 |
| [Broker Architecture](./ブローカーアーキテクチャ.md) | 仲介役のブローカーを介してコンポーネント間の要求やメッセージを配送する構成です。 |
| [Space-Based Architecture](./スペースベースドアーキテクチャ.md) | 分散キャッシュや共有データグリッドを中心に、処理ノードを水平に増やす構成です。 |
