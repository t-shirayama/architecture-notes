# レイヤー・責務分離系

層、依存方向、責務分離、ドメイン中心設計を扱うカテゴリです。

## 作成済みドキュメント

| アーキテクチャ | 概要 |
| --- | --- |
| [レイヤードアーキテクチャ](./レイヤードアーキテクチャ.md) | レイヤードアーキテクチャは、アプリケーションを責務ごとの層に分け、上位層から下位層へ処理を流す基本的な構成です。 |
| [クリーンアーキテクチャ](./クリーンアーキテクチャ.md) | クリーンアーキテクチャは、業務ルールを中心に置き、UI、DB、Webフレームワーク、外部サービスなどの詳細を外側へ追い出す設計方針です。 |
| [ヘキサゴナルアーキテクチャ](./ヘキサゴナルアーキテクチャ.md) | ヘキサゴナルアーキテクチャは、Ports & Adaptersとも呼ばれ、アプリケーションの内側を外部技術から分離する設計です。 |
| [オニオンアーキテクチャ](./オニオンアーキテクチャ.md) | オニオンアーキテクチャは、Domain Modelを中心に置き、依存関係を中心へ向ける設計です。 |
| [DDDアーキテクチャ](./DDDアーキテクチャ.md) | DDD、Domain-Driven Designは、複雑な業務領域をソフトウェアで扱うために、ドメイン知識を中心に設計する考え方です。 |
| [BCE](./BCE.md) | Boundary / Control / Entity に責務を分け、ユースケースの流れを中心に整理する設計です。 |
| [Vertical Slice Architecture](./バーティカルスライスアーキテクチャ.md) | 技術レイヤーではなく、機能やユースケース単位でコードを縦に分割する設計です。 |
| [Modular Architecture](./モジュラーアーキテクチャ.md) | 明確な責務と公開インターフェースを持つモジュールに分割し、変更影響を閉じ込める設計です。 |
