# レイヤー・責務分離系

層、依存方向、責務分離、ドメイン中心設計を扱うカテゴリです。

## 作成済みドキュメント

| アーキテクチャ | 概要 |
| --- | --- |
| [レイヤードアーキテクチャ](./レイヤードアーキテクチャ.md) | Presentation / Application / Domain / Infrastructure のように層で分ける基本形 |
| [クリーンアーキテクチャ](./クリーンアーキテクチャ.md) | 依存関係を内側へ向け、業務ルールを外部詳細から守る設計 |
| [ヘキサゴナルアーキテクチャ](./ヘキサゴナルアーキテクチャ.md) | PortとAdapterでアプリケーションと外部技術を分離する設計 |
| [オニオンアーキテクチャ](./オニオンアーキテクチャ.md) | Domain Modelを中心に置き、依存を中心へ向ける設計 |
| [DDDアーキテクチャ](./DDDアーキテクチャ.md) | 業務知識をモデル化し、言葉と境界を中心に設計する考え方 |
