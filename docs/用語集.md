# 用語集

## 概要

この用語集は、各アーキテクチャ文書で繰り返し出てくる言葉を短く確認するための補助資料です。厳密な定義は文脈によって変わるため、ここでは実務で読み替えやすい意味を中心に整理します。

## 基本用語

| 用語 | 意味 |
| --- | --- |
| 責務 | その要素が何を担当し、何を担当しないかという役割 |
| 境界 | 変更、所有、依存、通信、データの切れ目 |
| 依存方向 | あるコードやモジュールが、どの相手を知ってよいかという向き |
| 抽象化 | 具体的な技術や実装を隠し、必要な契約だけを見せること |
| 凝集度 | 関連するものが同じ場所にまとまっている度合い |
| 結合度 | ある要素が他の要素にどれだけ強く依存しているか |
| デプロイ単位 | 変更を本番へ反映するときに一緒にリリースされる単位 |
| 所有権 | ある機能、データ、API、運用責任をどのチームやモジュールが持つか |

## データと整合性

| 用語 | 意味 |
| --- | --- |
| 強整合性 | 更新後、関係する利用者や処理がすぐ一貫した状態を見られる性質 |
| 最終的整合性 | 一時的な不整合を許容し、時間が経つと一貫した状態へ収束する性質 |
| 冪等性 | 同じ操作を複数回実行しても結果が変わらない性質 |
| 補償処理 | 完了済みの処理を業務的に取り消すための処理 |
| Projection | イベントや更新結果から参照用のデータを作る処理 |
| Read Model | 参照や検索に最適化したデータ表現 |
| Event | 業務上すでに発生した事実を表すデータ |
| Event Store | Event Sourcingでイベントを追記保存するストア |

## 分散システムと連携

| 用語 | 意味 |
| --- | --- |
| 同期通信 | 呼び出し元が応答を待つ通信 |
| 非同期通信 | 呼び出し元が処理完了を待たず、キューやイベントで連携する通信 |
| Broker | 送信側と受信側の間でメッセージや要求を仲介する要素 |
| Topic | Publish-Subscribeでメッセージを分類する単位 |
| Queue | メッセージを順番に保持し、Consumerへ渡す待ち行列 |
| Dead Letter Queue | 処理に失敗し続けたメッセージを隔離するキュー |
| API Gateway | 複数サービスへのAPI入口を集約する層 |
| BFF | 特定のフロントエンド向けに最適化されたバックエンド層 |

## 運用と信頼性

| 用語 | 意味 |
| --- | --- |
| Observability | ログ、メトリクス、トレースからシステム状態を把握できる性質 |
| Rollback | 問題発生時に以前の状態へ戻すこと |
| Roll Forward | 問題を修正版の追加リリースで前に進めて解決すること |
| Guardrail | 自動化やリリースを止めるための安全条件 |
| Timeout | 応答を待つ最大時間 |
| Retry | 失敗した処理を再試行すること |
| Circuit Breaker | 失敗が続く呼び出しを一時遮断する仕組み |
| Bulkhead | リソースや処理を隔離し、障害波及を抑える考え方 |

## フロントエンド

| 用語 | 意味 |
| --- | --- |
| State | UIやアプリケーションが現在持っている状態 |
| Store | 共有状態を保持する場所 |
| Action | 状態変更の意図や出来事を表すデータ |
| Reducer | 現在状態とActionから次状態を作る関数 |
| ViewModel | Viewに必要な状態や操作を表すモデル |
| Hydration | サーバー生成HTMLにクライアント側JavaScriptの振る舞いを接続すること |
| Component | UIを構成する再利用可能な部品 |
