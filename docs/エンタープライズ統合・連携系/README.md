# エンタープライズ統合・連携系

複数システムの連携、メッセージング、段階的移行、統合基盤を扱うカテゴリです。

## 作成済みドキュメント

| アーキテクチャ | 概要 |
| --- | --- |
| [Pipe and Filter](./パイプアンドフィルター.md) | 処理を独立したFilterに分け、Pipeでデータを順に流す構成です。 |
| [Publish-Subscribe](./パブリッシュサブスクライブ.md) | 発行者がメッセージをTopicへ発行し、購読者が関心のあるTopicを受け取る通信パターンです。 |
| [Message Queue Architecture](./メッセージキューアーキテクチャ.md) | 送信側と処理側の間にキューを置き、メッセージを非同期に受け渡す構成です。 |
| [ESB](./ESB.md) | 企業内の複数システムをバス型の統合基盤で接続し、ルーティングや変換を担う構成です。 |
| [API Gateway Pattern](./APIゲートウェイパターン.md) | 複数サービスへの入口を集約し、認証、ルーティング、レート制限、集約などを担わせるパターンです。 |
| [Strangler Fig Pattern](./ストラングラーフィグパターン.md) | 既存システムの周囲に新しい経路を少しずつ作り、段階的に古い機能を置き換える移行パターンです。 |
